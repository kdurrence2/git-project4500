Installing Git

  1. Navigate to https://git-scm.com/ in your browser.
  2. Navigate to the Git Setup File Download
    • Select the “Download for [OS]” button on the lower right hand side of the screen to access the download link that matches your operating system.
    • The browser will detect the correct operating system and the appropriate download file will be the default option.
  3. Select the Download link
    • The latest release version download link is at the top of the page.
  4. Access the Setup File from Downloads
    • In the top right corner of your browser, select the downloaded icon to access your recent downloaded files. Select the Git Setup file to begin the installation.
  5. Complete the Git Setup Process
    • Complete all initial set up steps to finish the installation.
    • Read each option carefully and make changes if necessary or use the default option.
  6. Git Has Been Successfully Installed!
    • You can now use Git version control functions in your projects!
  7. NOTE: Setting The Git Default Editor
    • Git will ask you to choose a default editor application. For the purposes of this tutorial, we will be utilizing Visual Studio Code (VS Code) as the default code editor. This option will require VS Code to be downloaded outside of this installer.
    • If you have not already downloaded VS Code, you can select the link in the menu and download the editor. VS Code Download Instructions
    • You will need to close the Git Setup Wizard, download Visual Studio Code, then reopen the Git Setup Wizard again. The setup wizard can be found in File Explorer → Downloads.

Installing Visual Studio Code

  1. Navigate to https://code.visualstudio.com// in your browser.
  2. Select the “Download for (OS)” Button
    • The browser will detect the correct operating system and the appropriate download file will be the default option.
  3. Access the File from Downloads
    • In the top right corner of your browser, select the downloaded icon to access your recent downloaded files. Select the VS Code Setup file to begin the installation.
  4. Complete the VS Code Setup Process
    • Complete all initial set up steps to finish the installation.
    • Read each option carefully and make changes if necessary or use the default option.
  5. Visual Studio Code Has Been Successfully Installed!
    • You can now start using VS Code for code editing!
  6. Optional: Complete the VS Code Walkthrough
    • This internal setup and walkthrough will help you personalize your experience and get more familiar with the program.
  7. Recommended: Install Code Language Extensions
    • Code Language Extensions will provide auto-complete suggestions, syntax assistance, and debugging options. Some languages come preset and others will need to be downloaded as extensions.
    • You can download these extensions during the walkthrough or any time from the action bar on the left hand side by clicking on the “Extensions” icon.

Configuring Git

Before jumping into the code, Git will need to be configured with author information. This information is used with each change and commit within the code files. The essential information is your name and email address.

  1. Access the Terminal Under the “View” Tab
    • While the new local repository is still open, access the “View” tab then select “Terminal.”
  2. Open Git Bash in the Terminal
    • Near “powershell” and “+” symbol, select the dropdown arrow and select “Git Bash.”
    • Git Bash is the terminal for Git command lines. The commands needed to set the user username and email address will need to be run in Git Bash to take effect.
  3. Set Your Username and Email Address
    • Use the $ git config --global user.name "ENTER YOUR NAME" command to set your user name and “$ git config --global user.email johndoe@example.com ” to set your email address.
    • Using the - - global option means that this information will be used for any changes made within the system.
  4. NOTE: How to Check These Settings
    • You can check that these settings were updated by entering $ git config user.name and $ git config user.email in the terminal.

Basic Git Operations

Here is how to perform basic Git operations through the VS Code terminal.

  1. Add
    • Git Command: In the terminal , the git add command will add content/changes from the current file to the index (the staged area that is ready for commit.)
    • VS Code GUI: Select the plus sign “+” on the file with untracked changes (marked with a U).
    • Once the changes are added (staged), you will see the file has been marked with an “A” and moved to the “Staged Changes” section.
  2. Status
    • Git Command: The git status command will show you a list of files that have changes and are staged (added) for the next commit.
  3. Commit
    • Git Command: The git commit command will record added/staged changes made in the index to the repository.
    • VS Code GUI: Select the “Commit” button in the “Source Control” panel. A message can be added in the text input field above to describe the commit and the changes that were made.
  4. Push
    • Git Command:The git push command will upload all local branch commits to GitHub/the remote repository.
    • VS Code GUI: Select “Publish Branch” and the local repository will upload to GitHub.
    • The changes can be seen on the GitHub repository.
  5. Pull
    • Git Command: The git pull command will update the current local working branch with any new updates from the corresponding remote branch on GitHub.
    • This command downloads all changes from remote branches (fetch) and combines remote branches changes into the current branch (merge).
    • VS Code GUI: Select the menu (…) next to the “Source Control” Header and select “Pull”
